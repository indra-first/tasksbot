# Быстрый задачник

## Описание

**Быстрый задачник** — это Telegram-бот-планировщик задач, который живёт там, где вы уже работаете: прямо в Telegram. Не нужно переключаться между приложениями — просто напишите задачу и время, бот сохранит её и напомнит вовремя.

## Какую задачу решает

Многие специалисты — руководители, разработчики, маркетологи — проводят значительную часть рабочего дня в Telegram. Открывать отдельный планировщик, заходить в стороннее приложение, вести ещё один сервис — это лишнее трение. Быстрый задачник устраняет это: задачи ставятся прямо из чата, без выхода из мессенджера.

**Проблемы, которые решает бот:**
- Разрозненность задач по разным инструментам
- Забытые дедлайны при плотном графике
- Необходимость переключаться между приложениями
- Отсутствие утреннего и вечернего обзора дня

## Для кого предназначен

- **Руководители команд и отделов** — быстро ставить себе задачи между встречами
- **Разработчики** — фиксировать технические задачи и дедлайны прямо из рабочего чата
- **Маркетологи** — контролировать публикации, дедлайны кампаний, встречи с клиентами
- **Все, кто много работает в Telegram** и не хочет вести задачи отдельно

## Возможности бота

| Функция | Описание |
|---|---|
| Добавление задач | Пишите задачу естественным языком с датой и временем |
| Напоминания | Автоматически за 10 минут до дедлайна (настраивается) |
| Повторяющиеся задачи | Ежедневно, еженедельно, раз в 2 недели, ежемесячно |
| Список задач | Активные задачи, на сегодня, на неделю |
| Управление | Отметить выполненной, удалить, отредактировать |
| Утренний дайджест | Сводка задач на день в 09:00 |
| Вечерний дайджест | Задачи на завтра в 21:00 |
| Кастомная сводка | Настройка времени дайджеста через `/setdigest` |
| История | Список выполненных задач |


## Команды

```
/start        — приветствие и справка
/help         — справка по командам
/list         — все активные задачи
/today        — задачи на сегодня
/week         — задачи на ближайшие 7 дней
/done <id>    — отметить задачу выполненной
/del <id>     — удалить задачу
/edit <id> <новый текст + время> — изменить задачу
/history      — история выполненных задач
/setdigest 08:30 — установить время утренней сводки
/setdigest off   — отключить сводку
```

## Как использовался Cursor

Проект создан с помощью **Cursor IDE** — ИИ-редактора нового поколения:

1. **Генерация структуры проекта** — архитектура разбита на модули (`bot.py`, `db.py`) с помощью Cursor Agent
2. **Написание парсера задач** — логика распознавания дат на русском языке (`parse_task`, регулярные выражения) сгенерирована и отлажена через чат Cursor
3. **Разработка базы данных** — схема SQLite, миграции и все функции модуля `db.py` написаны при помощи Cursor
4. **Правила кодирования (Rules)** — созданы `.cursor/rules/python-bot.mdc` с правилами логирования, типизации и обработки ошибок, которым следует весь проект
5. **Отладка и рефакторинг** — ошибки в планировщике задач (job queue), timezone-aware datetime, восстановление задач после перезапуска — всё исправлено через Cursor
6. **Документация** — этот README написан с помощью Cursor

## Как запустить

### 1. Требования
- Python 3.11 или выше
- Telegram-аккаунт и бот, созданный через [@BotFather](https://t.me/BotFather)

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка токена

Скопируйте `.env.example` в `.env` и вставьте токен вашего бота:

```bash
cp .env.example .env
```

Откройте `.env` и замените `your_telegram_bot_token_here` на реальный токен:

```
BOT_TOKEN=ваш_токен_от_BotFather
```

### 4. Запуск

```bash
python bot.py
```

После запуска найдите своего бота в Telegram и напишите `/start`.

## Необходимые библиотеки и настройки

```
python-telegram-bot[job-queue]==21.9   — основная библиотека для Telegram-ботов
python-dotenv==1.0.1                   — загрузка переменных окружения из .env
```

База данных SQLite (`tasks.db`) создаётся автоматически при первом запуске — ничего дополнительно устанавливать не нужно.

### Важно
- Файл `.env` с реальным токеном **не входит в архив**. Используйте `.env.example` как шаблон.
- Бот хранит данные в `tasks.db` (SQLite). При переносе на другой сервер скопируйте этот файл или запустите с нуля — база создастся заново.
- Часовой пояс по умолчанию: **Europe/Moscow** (можно изменить в `bot.py`, строка `LOCAL_TZ`).
