# Быстрый задачник

## Описание

**Быстрый задачник** — Telegram-бот-планировщик, который живёт там, где вы уже работаете: прямо в Telegram. Не нужно переключаться между приложениями — просто напишите задачу и время, бот сохранит её и напомнит вовремя.

## Какую задачу решает

Многие специалисты — руководители, разработчики, маркетологи — проводят значительную часть рабочего дня в Telegram. Открывать отдельный планировщик — лишнее трение. Быстрый задачник устраняет это: задачи ставятся прямо из чата.

## Для кого предназначен

- **Руководители** — быстро ставить себе задачи между встречами
- **Специалисты по продажам и BDM** — трекать follow-up переговоров
- **Все, кто много работает в Telegram** и не хочет вести задачи отдельно

## Возможности

| Функция | Описание |
|---|---|
| Задачи на естественном языке | «Завтра в 10 врач» / «Звонок в пятницу в 3 дня» |
| Напоминания | «напомни за 30 минут» / «напомни в 13:30» |
| Кнопка Отложить | +30 мин / +2 часа / Завтра 9:00 прямо из уведомления |
| Повторяющиеся задачи | каждый день / неделю / 2 недели / месяц |
| Приоритеты | 🔴 Срочно / 🟡 Важно — через кнопки или слово в тексте |
| Категории | Работа / Семья / Личное — через кнопки или #хештег |
| Просроченные | /overdue — список с кнопками переноса |
| Утренний дайджест | Сводка на день в 09:00, с пометкой просроченных |
| Вечерний дайджест | Задачи на завтра в 21:00 |
| Семейный список | /family — совместные задачи с партнёром |
| История | Список выполненных задач |

## Примеры использования

```
Завтра в 10 утра врач напомни за 30 минут
Звонок партнёру в пятницу 15:00 срочно #работа
Купить торт для сына послезавтра в 6 вечера #семья
Отчёт каждую неделю пятница 17:00 важно
Встреча 12.06 14:00 напомни в 13:30
Таблетки через 2 часа #личное
```

После ввода задачи без категории бот предложит выбрать через кнопки:

```
📂 Выбери категорию:
[💼 Работа] [👨‍👩‍👦 Семья] [👤 Личное] [⏭ Пропустить]
```

Затем приоритет:

```
🎯 Выбери важность:
[🔴 Срочно] [🟡 Важно] [⚪ Обычная]
```

## Команды

```
/start           — приветствие и справка
/help            — справка
/list            — все активные задачи
/today           — задачи на сегодня
/week            — задачи на неделю
/overdue         — просроченные задачи
/done <id>       — отметить выполненной
/del <id>        — удалить задачу
/edit <id> <...> — изменить задачу
/history         — история выполненных
/family          — семейный список (создать/принять приглашение)
/family unlink   — отключить семейный список
/setdigest 08:30 — время утренней сводки
/setdigest off   — отключить сводку
```

## Деплой на Railway

### 1. Форкни или загрузи репозиторий на GitHub

### 2. Создай проект на Railway

1. Зайди на [railway.app](https://railway.app) → **New Project**
2. Выбери **Deploy from GitHub repo** → выбери свой репозиторий
3. Railway автоматически обнаружит Python и `Procfile`

### 3. Добавь переменную окружения

В разделе **Variables** добавь:

```
BOT_TOKEN = ваш_токен_от_BotFather
```

### 4. Деплой

Нажми **Deploy** — Railway соберёт и запустит бота.
Статус смотри во вкладке **Deployments** → **View Logs**.

> **Важно:** SQLite (`tasks.db`) хранится в памяти контейнера. При рестарте Railway данные сохраняются, если используется постоянный том (Volume). Для личного бота это приемлемо — данные живут всё время работы сервиса без перезапуска.

## Локальный запуск

### Требования
- Python 3.11+
- Telegram-бот, созданный через [@BotFather](https://t.me/BotFather)

### Установка

```bash
pip install -r requirements.txt
```

### Настройка

```bash
cp .env.example .env
# Открой .env и вставь токен:
# BOT_TOKEN=ваш_токен_от_BotFather
```

### Запуск

```bash
python bot.py
```

## Как использовался Cursor

Проект создан с помощью **Cursor IDE**:

1. **Генерация архитектуры** — структура `bot.py` / `db.py`, парсер дат на русском языке
2. **Отладка парсера** — исправление багов с "завтра в 10 утра", часовые пояса, edge-cases
3. **Новые функции** — приоритеты, категории, снуз, просроченные, семейный список — всё через Cursor Agent
4. **Rules** — `.cursor/rules/python-bot.mdc` с правилами логирования, типизации, обработки ошибок
5. **Документация** — этот README

## Структура проекта

```

    └── rules/
        └── python-bot.mdc
```
