### Урок 12. Финальный проект

#### Задание:

```
Создать собственный рабочий мини-проект, в котором вы
используете Cursor для решения реальной задачи из своей
профессиональной деятельности.
Главная задача проекта — показать, что вы умеете применять
Cursor для автоматизации, ускорения или улучшения
рабочих процессов.
Проект может быть связан с любой сферой деятельности:
управление, маркетинг, аналитика, образование, разработка,
административная работа и т.д.
```
## Описание задания

```
Вам необходимо придумать и реализовать небольшой
рабочий инструмент или сценарий , созданный с помощью
Cursor.
Это может быть:
● скрипт
● чат-бот
● шаблон автоматизации
● генератор документов
● небольшой сервис
● инструмент обработки данных
● инструмент для подготовки текстов или отчётов
Главное условие — проект должен решать конкретную
практическую задачу.
```


    **помощью Cursor**.

       ● генерация кода


```

● написание функций

```
**2. Практическая ценность**
    Проект должен:

       ● быть **применимым в работе**
       ● иметь **понятное назначение
3. Минимальная сложность проекта**
    Проект должен включать **как минимум один из следующих**

       ● простой сервис или инструмент

       ● генератор документов
**4. Использование возможностей Cursor**
    В проекте желательно применить хотя бы один из следующих
    инструментов:
       ● Rules
       ● Skills
       ● Sub-agents
       ● генерацию кода через чат Cursor

#### 3. Что может снизить оценку?

```
● Непонятно, как запустить проект
В README нужно написать, какой файл открыть или какую
команду выполнить.
● Не описано использование Cursor
Так как это итоговый проект по Cursor, нужно показать, как
Cursor помогал при разработке.
● В архив попали личные токены, пароли или API-ключи
Не загружайте реальные ключи, токены, пароли и файл .env.
Вместо этого добавьте .env.example без секретных данных.
● В архиве есть лишние технические папки
Не нужно загружать .venv, venv, __pycache__,
node_modules, .git, .DS_Store и похожие файлы.
```
